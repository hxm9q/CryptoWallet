# CryptoWallet

Реализация приложения **CryptoWallet** на **Swift** с использованием **UIKit и SnapKit**.

**Приложение отображает список криптовалют с их текущими ценами, показателем изменения цены. А также предоставляет детальную информацию о монетах на отдельном экране.**

# Архитектура

**Архитектура проекта реализована на основе архитектурного паттерна **MVVM** с разделением ответственности по слоям и модулям.**
**Масштабируемая структура проекта: добавление новых элементов не требует изменений в логике.**

**Архитектура состоит из следующих компонентов:**

- **Models** - описывают данные (`Coin`, `CoinData`)
- **View** - UI-слой на UIKit (`ViewController` + Custom Cells)
- **ViewModel** - состояние, бизнес-логика, данные для отображения (`ViewModel`)
- **Service** - изолированный слой работы с сетью

# Ключевые решения

## Экран авторизации

- Реализовано сохранение состояния с использованием `UserDefaults`
- Переход на экран списка монет реализован не через push, а путем смены рутового `ViewController`'а у `UINavigationController`.
- Адаптивный layout для iPhone с экранами разного размера
- Все элементы расположены внутри `UIScrollView`, что позволяет избегать перекрытия полей ввода всплывающей клавиатурой. При показе клавиатуры увеличивается отступ внизу `scrollView`.
- Используются делегаты `UITextFieldDelegate`, чтобы при нажатии на клавиатуре клавиши "Далее" фокус переходил ко второму полю, а при "Готово" — происходила попытка авторизации.
- Также реализована функция сокрытия клавиатуры по тапу в любом месте экрана.
- При неудачной попытке входа пользователю предлагается повторить или очистить поля.
- Если пользователь оставил одно из полей пустым показывается алерт, сообщающий ему об этом.

## Экран списка криптовалют

- Приложение получает список нужных монет от API и показывает их в `UITableView`
- Адаптивный layout для iPhone с экранами разного размера
- При нажатии на функциональную кнопку показывается всплывающее меню, в котором пользователь может выйти и вернуться на экран авторизации или обновить данные о монетах.
- При загрузке и обновлении данных показывается индикатор загрузки.
- Добавлена кнопка сортировки по возрастанию или убыванию цены монеты, при нажатии на нее показывается всплывающее меню.
- Реализована плавная анимация смены данных при сортировке.
- Так как API не предоставляет изображения монет, добавил логотипы для нужных монет в Assets для демонстрации интерфейса.
- Снизу добавлен `TabBar` с заглушками на остальных `View`.
- При нажатии на ячейку с монетой пользователь переходит на отдельный экран с подробной информацией.

## Экран детальной информации о выбранной монете

- Отображение детальной информации о монете с новыми параметрами capitalization и supply.
- Адаптивный layout для iPhone с экранами разного размера.
- Кастомная кнопка возвращения на предыдущий экран.
- Используется `UISegmentedControl` для выбора временного диапазона (1D, 1W, 1Y, ALL, Point) — данные подставляются динамически из `ViewModel`.
- Поскольку при запросе данных за различные временные периоды API возвращал `null`, в проект была добавлена тестовая монета с демонстрационными значениями. Это позволило проверить и убедиться, что интерфейс и логика корректно обрабатывают неполные или отсутствующие данные, обеспечивая стабильную работу приложения.
